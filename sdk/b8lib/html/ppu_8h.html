Detailed Description

Picture Processing Unit (PPU) definitions for the BEEP-8 system.

This file contains the definitions and function prototypes for the Picture Processing Unit (PPU) in the BEEP-8 system. It includes register definitions, command codes, color definitions, and structures for various PPU operations such as drawing rectangles, polygons, sprites, and handling background tiles.

The PPU is responsible for rendering graphics and handling various graphical operations in the BEEP-8 system. It supports a range of commands for different drawing and rendering tasks, as well as configurations for color palettes, view offsets, and scissoring.

For more detailed information, please refer to the BEEP-8 data sheet.

Adds a primitive to the Ordering Table (OT) at the specified Z-value.

This function inserts a primitive command into the Ordering Table at the position corresponding to the given Z-value (otz_). The OT manages the order of rendering commands based on their Z-values, allowing proper depth sorting for rendering.

Parameters
cmd_A pointer to the PPU command structure.
otz_The Z-value at which the primitive should be inserted in the OT.
prim_A pointer to the primitive command to be added to the OT.
Returns
int Returns 0 on success, or a negative error code if the Z-value is invalid.

If the specified Z-value (otz_) is greater than or equal to the number of entries in the OT, the function will return an error code indicating an invalid value. Otherwise, it inserts the primitive into the OT, updating the jump commands accordingly.

Allocates and initializes an Ordering Table (OT) for the PPU commands.

This function allocates memory for an Ordering Table and sets up the necessary jump commands and halt command to create a linked list structure for PPU commands. The OT is used to manage the order of rendering commands based on their Z-values.

Parameters
cmd_A pointer to the PPU command structure.
ot_A pointer to the memory allocated for the Ordering Table.
num_The number of entries in the Ordering Table.

The function will initialize the OT entries with jump commands that link each entry to the previous one, with the last entry linked to a halt command. This setup allows the PPU to process commands in the correct order based on their Z-values.

Example:

cmd_->buff[ 0 ] -> HALT
cmd_->buff[ 1 ] -> &cmd_->buff[ 0 ]
cmd_->buff[ 2 ] -> &cmd_->buff[ 1 ]
cmd_->buff[ 3 ] -> &cmd_->buff[ 2 ]
...
cmd_->buff[ num_ - 1 ] = B8_PPU_CMD_JMP to &cmd_->buff[ num_ - 2 ]
